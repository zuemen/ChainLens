# 決賽 Demo 演示腳本 — 50 萬 USDT 出金攔阻情境

> 對應提案書第一章招牌情境。全程離線可跑（內建劇本圖），現場無網路也不炸。
> 建議演示時間：2.5–3 分鐘（嵌入 8 分鐘簡報的中段）。

## 演示前準備

```bash
make app   # 或 streamlit run chainlens/app/workbench.py
```

- 模式選「出金審查 Demo（50 萬 USDT 攔阻情境）」
- 事先跑一次確認瀏覽器快取熱好，現場按鈕即出結果

## 演示動線（口白 ＋ 操作）

### 第 1 步：拋出問題（口白，約 30 秒）

> 「某交易所用戶申請將 50 萬 USDT 提領到這個外部地址。這個地址**從來沒被通報過、
> 不在任何黑名單上**。傳統法遵工具比對名單——放行。錢一出去，就跨境追不回來了。」

### 第 2 步：執行審查（操作，約 20 秒）

- 目標地址保持預設 `TOtcOut01`、金額 500,000 → 按「執行出金審查」
- 指著決策卡念出：**綜合風險 0.77 → 暫緩出金並啟動人工審查**

> 「注意三個分數：它**自身**結構分數只有 0.42——名單與單點特徵都攔不住它。
> 真正拉高風險的是**關聯分數 0.60**：它的上游兩步之內，就是一個命中
> 『集資扇入＋快速分散＋集散』三種洗錢圖樣的集資主錢包。」

### 第 3 步：圖譜說故事（操作，約 60 秒）

- 捲到金流圖譜，指出橘色高亮路徑：`TAggregator01 → TMule03 → TOtcOut01`
- 由左至右講一次完整劇本：

> 「左邊 12 個被害人，在『限時加碼』話術下 50 分鐘內集中入金到 3 個客服收款地址
> ——這是 fan-in 集資。資金歸集到主錢包後，30 分鐘內拆給 6 個車手——fan-out。
> 車手再走剝洋蔥鏈層層轉手，最後整合到這個 OTC 出金地址。
> 集資—分層—整合，教科書級的洗錢三階段，我們的圖樣庫**主動掃出來**的，
> 不需要任何黑名單。」

### 第 4 步：STR 草稿收尾（操作，約 40 秒）

- 展開「可疑交易申報（STR）草稿」，捲動展示證據鏈

> 「而且每一筆判定都有可稽核的證據：哪個風險節點、幾階關聯、路徑上每一跳的
> 金額和時間戳。法遵人員審閱後可直接轉為申報文件——這就是我們說的
> 『調查敘事導向』，不是黑箱分數。」

- 點「下載 STR 草稿」展示一鍵產出

### 第 5 步：對照組防守（備用，約 20 秒）

評審若質疑誤報率，切換目標為 `TNormalUser01`（正常用戶）再按一次：

> 「同一套引擎，正常地址風險 0.20，放行。結構偵測不是寧可錯殺。」

## 預期 QA 與回應要點

| 可能提問 | 回應要點 |
|---|---|
| 資料是真的嗎？ | 劇本圖為合成資料，比例與時間節奏參照 165 公開案例之假投資出金模式；工作台另有「金流圖譜工作台」模式可抓 TRON 主網真實地址 2-hop 圖 |
| 閾值怎麼定？ | 0.7/0.4 為 Demo 預設；正式部署依業者風險胃納與歷史誤報率校準，並支援分層處置 |
| 和 GNN 的關係？ | 本 Demo 展示規則＋關聯層；GNN 分數以 noisy-or 併入同一融合公式（`screening.py`），Elliptic 基準見 RESEARCH.md |
| 誤報怎麼辦？ | 決策是「暫緩＋人工審查」而非直接拒絕；STR 草稿即為人工審查的起點，人在迴路中 |
